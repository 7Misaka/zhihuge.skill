## 1. 为什么先建矩阵

矩阵把“论文想说什么”与“现有证据允许说什么”分开。它用于在写作前发现三类断链：有组件但没有问题、有定理但没有可解释结论、有实验但没有对应主张。

每个核心主张只设一个主 ID。相同主张在摘要、引言和结论出现时复用该 ID，不创建重复版本。

## 2. 字段定义

| 字段 | 要回答的问题 |
|---|---|
| Claim ID | 该主张的稳定标识是什么？ |
| Claim | 可证伪的一句话是什么？ |
| Observed problem | 什么现象需要解释或改善？ |
| Causal mechanism | 为什么会出现该现象？ |
| Intervention/component | 方法实际改变什么？ |
| Theory object | 用什么量、界或复杂度刻画？ |
| Assumptions | 结论依赖哪些条件？ |
| Manipulated variable | 实验主动改变什么？ |
| Controls | 哪些因素必须保持不变？ |
| Metric | 用什么指标观察？ |
| Predicted direction | 理论或机制预测怎样变化？ |
| Evidence | 哪个定理、图、表或统计量支持？ |
| Boundary | 在什么条件下不成立或尚未知？ |
| Manuscript location | 在摘要、引言、方法、理论、实验、讨论何处出现？ |
| Status | supported / partial / missing / contradicted |

## 4. 填写流程

### 4.1 从观察开始

先写可观察问题，不先写方法名。例如写“拓扑变稀疏时达到同一精度需要更多轮次”，不要写“需要一种拓扑鲁棒方法”。

### 4.2 写机制而非形容词

把“低效”“不稳定”“泛化差”展开成变量关系。例如：局部更新造成节点漂移；较小谱隙减慢混合；异质分布扩大模型分歧。

### 4.3 定义干预

写组件实际改变的更新、约束、状态或通信行为。若只能写“增强特征”“提升鲁棒性”，说明组件尚未被解释清楚。

### 4.4 选择理论对象

理论对象必须接近主张：

- 速度主张：迭代/通信复杂度或达到阈值的界。
- 优化主张：梯度范数、次优性或一致性误差。
- 泛化主张：稳定性、泛化差距或超额风险。
- 网络主张：谱量、混合误差或共识项。
- 系统效率主张：通信字节、墙钟时间、内存或能耗。

### 4.5 派生实验

从理论项读取变量，写出方向性预测，再设计控制实验。不得先看到结果后倒写预测而不注明探索性分析。

### 4.6 限定主张

把假设、数据范围、模型规模、拓扑类型和预算写入边界。对 `partial` 主张降低措辞强度；对 `missing` 主张删除、补证据或明确为未来工作。

## 5. 三类主张的证据要求

### 5.1 方法新颖性

最低证据：与最近方法的操作级差异、组件必要性、不是简单重命名的说明。仅有更高结果不能证明新颖。

### 5.2 理论主张

最低证据：正式假设、定理对象、依赖关系、证明或可核查推导、与基线在相同条件下的比较。实验一致不能替代理论证明。

### 5.3 实证主张

最低证据：公平预算、强基线、重复和不确定性、多个相关条件、失败或无优势场景。单个数据集的最好点不能支持普适优越性。

### 5.4 机制主张

最低证据：操纵机制变量的受控实验、组件消融或中间量测量，再结合理论或因果解释。总体性能提升本身不能识别机制。

### 5.5 效率主张

最低证据：明确分母。分别报告每轮成本、达到目标的总成本和墙钟时间；避免只用轮数声称更高效。

填写规则：

1. `Bound term` 写出变量所在项的语义，不必复制完整公式。
2. `Variable` 只能填实验可控制或可测量的量。
3. `Directional prediction` 写增大/减小/饱和/非单调，不写“有影响”。
4. `Falsifying observation` 提前定义什么结果会与预测冲突。
5. 如果理论对象不可直接测量，明确代理指标和代理误差。

| ID | Claim | Problem | Mechanism | Component | Theory | Experiment | Boundary | Status |
|---|---|---|---|---|---|---|---|---|
| C1 | 对偶约束可减小去中心化局部模型的不一致影响 | 节点仅与邻居交互时模型持续分歧 | 局部更新和有限混合累积偏差 | ADMM 式对偶状态 | 收敛界中的共识/网络项 | 改变拓扑连通性并比较完整方法与无约束变体 | 依赖连通网络与分析假设 | supported |
| C2 | 平坦化局部更新可缓解异质数据上的局部过拟合 | 节点分布不同导致局部解偏向自身数据 | 尖锐局部极小值放大跨节点差异 | SAM 式扰动更新 | 若无直接泛化定理，则仅声称机制动机 | 异质性强度扫描和组件消融 | 不能由训练损失单独证明泛化改善 | partial |
| C3 | 增加 gossip 步数会通过更充分混合改善稳定性相关泛化项 | 单次混合保留更大网络误差 | 混合矩阵幂衰减非共识分量 | 多次 gossip | 稳定性/泛化界中的网络项 | 扫描 gossip 步数和谱量，观察饱和趋势 | 有限预算实验不能证明无穷极限等价 | supported |

## 8. 双向审计

### 8.1 从主张到证据

对每个 Claim ID 追问：

1. 哪个观察说明问题真实存在？
2. 哪条机制解释为什么组件会有用？
3. 哪个定理或实验支持主张？
4. 哪个结果可能否证它？
5. 哪些条件限制它？

任一项为空，将状态标为 `partial` 或 `missing`。

### 8.2 从证据到主张

对每个定理、图和表反向追问：

1. 它回答哪个 Claim ID 或 RQ？
2. 为什么必须放在正文？
3. 它与其他证据是否重复？
4. 如果删除，哪条主张会失去支持？

若无法回答，删除该证据、移入附录或重新定义其目的。

### 8.3 从句子到矩阵

摘要、贡献列表和结论中的每个强主张都必须能映射到矩阵。新出现的主张不能因为位置靠后而豁免证据要求。

## 9. 冲突与缺失处理

### 9.1 理论与实验方向冲突

依次检查：

1. 实验是否满足理论假设。
2. 被测量指标是否真是理论对象。
3. 有限样本、常数项或高阶项是否主导。
4. 是否存在两个同时变化的混淆变量。
5. 实现是否与算法定义一致。

不能解释时，保留冲突并降低结论，不筛掉不利结果。

### 9.2 缺少理论证据

将措辞从“保证”降为“旨在”“经验上观察到”或“与机制解释一致”，并在局限中说明缺口。

### 9.3 缺少实验变量

若定理依赖某变量但实验没有覆盖，将其列为缺失证据。不要使用另一个相关变量作为无声明替代。

### 9.4 来源间不一致

优先采用最新原始版本；记录版本和章节。若仍冲突，分别呈现，不把多数论文写成事实裁决。
