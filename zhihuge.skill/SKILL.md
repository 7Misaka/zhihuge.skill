# 智虎哥理论驱动论文框架

把论文写成一条可检验的因果链：**问题 → 机制 → 组件 → 定理 → 实验 → 边界**。不要先堆章节或术语；先锁定主张以及每项主张所需的证据，再展开文本。

## 核心原则

1. 只让一句话承担一个主要功能：定义、判断、因果、对比、证据或过渡。
2. 把宏观痛点拆成两个可由不同机制处理的子问题；每个方法组件只解决一个明确子问题。
3. 让理论揭示变量如何影响结果，而不只给出“收敛”或“可泛化”的结论。
4. 让实验直接操纵理论中的变量，并检验定理给出的方向性预测。
5. 把强假设、极端情形、反直觉发现和无效场景写进讨论或局限，不要藏在附录。
6. 区分来源事实、跨论文归纳和当前论文设计建议；不得把归纳写成原作者的直接主张。

三、可复用论文框架
0. 写作前的主张—证据矩阵
正文动笔前，每个核心贡献填写一行：
主张	问题现象	原因机制	方法组件	理论项	受控实验	边界
C1	什么指标恶化	为什么恶化	哪个组件处理	哪个定理项解释	改变什么变量	何时不成立
C2						
C3						

任何空白都代表论文尚未闭环：
有方法、无理论：只能写经验贡献。
有理论、无实验：不能宣称已得到经验验证。
有总体性能、无组件消融：不能声称某个组件导致提升。
只有上界：不能将上界写成实际性能的精确等式。
1. 标题
推荐结构：
方法型：[方法名]: [核心机制] for [任务或问题]
结果型：[动作性结果]: [分析方法] in [研究场景]
问题型：Why/Can [技术现象]? A [理论视角]
机制型：Unveiling/Understanding [技术机制]: [理论贡献]
标题必须同时出现：
研究对象；
核心机制或理论视角；
主要结果或问题。
2. Abstract：9—10 句
通用九句结构
领域价值句：说明研究范式解决什么现实约束。
现有方案句：说明当前主流方法如何工作。
失败现象句：指出现有方案在特定条件下出现什么可观测问题。
研究缺口句：说明该问题尚未被算法或理论真正解决。
方法总览句：提出方法或分析框架，并说明总体目标。
组件一机制句：写第一个组件处理哪个原因。
组件二机制句：写第二个组件处理哪个原因。
理论结果句：给出优化、稳定性或泛化结果，保留关键阶数和变量。
实验结果句：写数据集、场景、比较对象和最重要的定量结果。
边界或意义句，可选：理论分析型论文写无法消除的极限差距；算法型论文写实际部署意义。
摘要检查
摘要中的每个结果都必须能在正文定位到：
一个定理、推论或 Remark；
一张主表或主图；
一个明确的实验设置。
3. Introduction：六段，约18句
第1段：领域与范式，3句
定义研究领域及现实需求。
说明主流范式的工作方式和优势。
引出目标范式为什么值得研究。
第2段：问题的因果链，4句
写目标范式的第一个可观测问题。
解释导致该问题的中间机制。
写第二个问题及其机制。
说明两个问题最终如何影响性能、效率或泛化。
不要只写“性能较差”，应写成：
在 [条件] 下，[机制变量] 持续累积，导致 [中间现象]，最终使 [指标] 恶化。

第3段：研究缺口或研究问题，2—3句
总结已有方法仍未处理的具体部分。
明确提出研究问题一。
明确提出研究问题二。
算法型论文可以把问题写成设计目标；机制解释型论文最好直接使用问句。
第4段：方法与组件映射，4句
提出方法或理论框架。
说明组件一修改了哪个更新阶段、解决哪个问题。
说明组件二修改了哪个阶段、解决哪个问题。
解释两个组件为何互补，而不是简单堆叠。
第5段：证据预览，4句
概述理论对象：优化、稳定性、泛化或极限分析。
写最重要的理论结论以及关键变量。
概述实验范围和比较对象。
写最重要的定量提升或理论趋势验证。
第6段：贡献列表
每条贡献采用同一格式：
做了什么；
解决哪个缺口；
通过什么机制；
由什么理论或实验支撑。
推荐顺序：
方法或分析框架贡献；
理论和机制贡献；
实验与实际指导贡献；
边界或极限结论，可选。
4. Related Work：三条研究线
开头用一句话声明将回顾哪些研究线。
每条研究线采用六句模板：
定义该研究线与本文问题的关系。
介绍早期代表方法解决了什么。
介绍近期方法如何改进。
指出最接近本文的方法及其机制。
分析这些方法仍依赖什么假设或遗漏什么变量。
用缺口句结束，并把缺口连接到本文贡献。
推荐三条线：
任务或范式：DFL、分布式优化等；
方法机制：ADMM、SAM、Momentum、Catalyst、MGS等；
理论工具：收敛、稳定性、泛化、拓扑分析等。
不要按作者年份流水账排列。每个小节都必须以“尚缺什么”结束。
5. Background / Problem Setup：10句
这部分可以独立成章，也可以作为 Methodology 的第一小节。
定义参与方、迭代轮数和局部步骤。
定义数据分布及异质性来源。
定义通信图、邻居集合和混合矩阵。
定义总体或 population objective。
定义 empirical objective。
定义全局最优值、经验最优值和算法输出。
定义需要分析的误差量。
给出基线算法的核心更新式。
指出基线更新式中产生问题的具体项。
说明后续方法或分析将如何处理该项。
理论分析型论文还应增加：
稳定性定义；
泛化误差与优化误差分解；
将稳定性连接到泛化误差的桥接引理。
6. Methodology
6.1 章节导语：2句
说明本节先定义什么、再设计什么。
说明完整方法由哪几个组件组成。
6.2 每个组件：七句单元
对组件一和组件二分别写：
失败定位：基线中的哪个步骤导致什么问题。
设计目标：本组件需要控制什么量。
数学修改：给出修改后的目标函数或更新式。
符号解释：解释新增变量、正则项或控制量。
机制解释：说明它如何改变优化轨迹或模型状态。
问题回链：指出它对应引言中的哪个问题。
代价与边界：说明新增计算、通信、超参数或假设。
多组件关系再写三句
单独使用组件一能解决什么，不能解决什么。
单独使用组件二能解决什么，不能解决什么。
两者联合为何互补，并对应哪个组合消融实验。
6.3 完整算法：4句
说明算法输入、输出和初始化。
说明每轮局部更新的执行顺序。
说明通信、聚合或外推顺序。
指明关键公式分别对应算法的哪些行。
7. Theoretical Analysis
7.1 章节导语：3句
本节分析哪个算法输出和误差。
先给哪些定义与假设。
随后给哪些定理，以及证明放在哪里。
7.2 每个假设：三句
正式写假设。
解释该假设控制什么现象。
说明它是否标准、能否放松，以及后续哪里使用。
不要把“有界梯度”和“有界随机梯度噪声”混为一谈。
7.3 分析难点：4句
标准分析为何不能直接使用。
新算法中的哪个项破坏了标准递推。
引入什么辅助序列、稳定性量或桥接条件。
该工具如何把复杂更新转成可分析对象。
7.4 桥接引理：4句
引理连接哪两个量。
写正式结论。
解释右侧各项的含义。
说明控制哪个量即可得到最终结果。
7.5 每个主定理：八句
写适用假设。
写参数范围。
写定理结论。
拆解主界中的优化项。
拆解异质性、噪声或拓扑项。
给出简化阶数。
翻译成参数变化预测。
指向验证该预测的实验小节。
7.6 每个 Remark：五句
明确讨论哪个变量。
从定理中提取包含该变量的项。
判断该变量变化造成的方向。
给出实际参数选择或系统设计建议。
写明该结论的条件或非单调边界。
7.7 极限与比较：六句
令关键变量趋于极端值。
写极限界。
写理想方法或中心化方法的界。
比较二者关于样本、节点或轮数的标度。
判断差距能否被单一机制消除。
把该边界加入摘要、讨论和结论。
8. Experiments
8.1 实验导语：3句
说明要验证的三类主张。
说明比较范围和主要场景。
指出附录包含哪些实现细节。
8.2 Experiment Setup：12句
数据集及其任务。
模型架构。
IID 设置。
非 IID 或困难程度构造。
客户端数量。
参与率。
通信拓扑。
中心化基线。
去中心化或同类基线。
训练轮数、批量和学习率。
方法特有超参数。
计算量和通信量的公平性说明。
8.3 主性能结果：七句
指出表格回答的研究问题。
陈述整体结果。
写最重要的绝对数值。
写相对最强基线的提升。
比较不同问题强度下的变化。
解释结果与哪个机制一致。
回链到贡献和主张编号。
8.4 收敛或效率：六句
定义固定性能阈值。
报告达到阈值所需轮数或时间。
计算相对加速比。
说明是否同时节省通信或计算。
对照理论中的轮数或参数项。
说明速度提升是否牺牲最终性能。
8.5 环境或拓扑实验：六句
定义各环境或拓扑条件。
说明它们改变了哪个理论量。
报告不同条件下的性能。
判断是否符合理论预测。
讨论反直觉结果。
给出部署层面的建议。
8.6 每个超参数消融：七句
说明参数在算法中的作用。
从定理中指出它影响的项。
写扫描范围。
说明其他变量保持不变。
报告最佳区间。
解释过小和过大分别为何失败。
判断实验是否支持理论预测。
8.7 组件消融：六句
定义完整模型。
移除所有新增组件。
仅保留组件一。
仅保留组件二。
同时使用两个组件。
判断二者是互补、冗余还是存在交互。
8.8 理论趋势验证：七句
选择能代理理论对象的观测量。
说明如何构造受扰动或配对数据。
改变一个理论关键变量。
绘制经验趋势。
与理论阶数或方向比较。
报告不吻合的区域。
说明这种验证支持的是趋势，不是定理证明。
9. Discussion：8句
提出最重要的反直觉发现。
说明已有理论通常给出什么结论。
说明本文理论为何不同。
给出相应实验观察。
区分收敛性能与泛化性能。
解释关键超参数为何有效。
说明极限情况和不可消除的差距。
给出实际使用建议和开放问题。
当理论和实验表面冲突时，Discussion 不能省略。
10. Conclusion and Limitation：8句
重述研究问题，而不是重复背景。
概括方法或分析框架。
说明组件一解决什么。
说明组件二解决什么。
概括最重要的理论结论。
概括最重要的实验结论。
写理论假设、规模或未验证条件。
给出具体未来方向。
结论不能引入正文未验证的新贡献。
11. Appendix
建议顺序：
实验实现和超参数；
通信与计算配置；
补充曲线和数据集结果；
符号表；
关键引理；
优化定理证明；
稳定性或泛化证明；
批量、拓扑或极限扩展分析；
额外局限性讨论。
四、最终复用原则
这个作者群体最值得复用的写作 Skill 可以压缩成五条：
引言中的问题数量，应与方法组件的逻辑数量大致对应。
每个理论变量都应转化成一项实验干预。
每项实验都要能回链到引言中的一个主张。
理论贡献不只给阶数，还要给参数选择、极限和不可达结论。
摘要和结论只报告已经进入“主张—机制—理论—实验”闭环的内容。
